Git Workflow

  • 1. Clone the Repository → git clone
  • 2. Create a Feature Branch → git checkout -b feature-name
  • 3. Make Code Changes
  • 4. Stage Changes → git add .
  • 5. Commit Changes → git commit -m "message"
  • 6. Push Branch → git push origin feature-name
  • 7. Create Pull Request (PR)
  • 8. Code Review & Merge into Main Branch

CI/CD Pipeline Stages

  • 1. Source Stage – Pipeline starts when code is pushed.
  • 2. Build Stage – Project is built (e.g., React build, Java compile).
  • 3. Test Stage – Unit tests, linting, automation tests run.
  • 4. Deploy to Staging – Code deployed to testing/staging server.
  • 5. Manual/Auto Approval
  • 6. Deploy to Production – Final deployment for real users.
  • 7. Monitoring – Logs, uptime checks, error tracking.
